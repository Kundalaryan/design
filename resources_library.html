Do / Don't Engineering Division UPDATED MAR 5

AI Tool Usage Rules — Engineering

Quick reference boundaries for using Large Language Models while coding, debugging, or reviewing architecture.

DO this

Use Acme Corp Enterprise LLM instances for any queries related to internal architecture or logic. These instances are zero-retention.
Obfuscate code before pasting into external consumer tools (like standard ChatGPT). Rename specific variable names to generic var_a, function_b().
Rigorously test AI-generated code snippets. Treat LLM output like code from a junior intern. Run unit and integration tests before committing.

NEVER do this

Paste proprietary source code containing core trade algorithms or API keys into public AI models to "check for bugs".
Ask an LLM to evaluate third-party library security. Models often hallucinate package safety. Always use Snyk or Checkmarx.
Upload entire stack traces containing user IDs or server IPs. Scrub logs locally before seeking assistance.
